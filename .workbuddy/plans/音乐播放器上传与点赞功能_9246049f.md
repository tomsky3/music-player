## 产品概述

为现有的个人音乐作品展示平台增加上传歌曲和点赞功能，让用户能够上传自己的音乐作品，并让访客可以点赞喜欢的歌曲。

## 核心功能

### 1. 上传歌曲功能

- 用户可选择本地MP3文件上传
- 上传时可填写歌曲信息（标题、艺术家、专辑）
- 可选择或上传封面图片
- 上传进度显示
- 上传成功后歌曲自动出现在播放列表中

### 2. 点赞功能

- 每首歌曲显示点赞数量
- 用户点击爱心图标进行点赞/取消点赞
- 点赞状态持久化保存（同一浏览器记住用户点赞记录）
- 点赞数实时更新

## 技术栈

### 前端

- **框架**：原生HTML/CSS/JavaScript（保持现有架构）
- **样式**：现有CSS变量系统 + Tailwind CSS
- **存储SDK**：腾讯云COS JavaScript SDK（cos-js-sdk-v5）
- **后端SDK**：Supabase JavaScript Client

### 云服务

- **腾讯云COS**：存储上传的MP3文件和封面图片
- 免费额度：50GB存储 + 10GB外网下行流量/月
- 支持前端直传（临时密钥STS方案）
- 国内访问速度快，无需翻墙

- **Supabase**：存储点赞数据
- 免费额度：500MB数据库 + 1GB文件存储 + 50,000月活用户
- PostgreSQL数据库
- 支持实时订阅

## 实现方案

### 上传功能架构

```
用户选择文件 → 前端验证(格式/大小) → 获取临时密钥 → 直传COS → 保存歌曲信息到Supabase → 刷新播放列表
```

### 点赞功能架构

```
用户点击点赞 → 检查本地点赞记录(LikeStore) → 更新Supabase点赞数 → 更新UI显示
```

### 安全考虑

- 使用临时密钥(STS)上传，避免暴露永久密钥
- Supabase使用RLS(Row Level Security)保护数据
- 限制上传文件类型和大小（MP3最大20MB，图片最大5MB）

## 关键代码结构

```javascript
export const CONFIG = {
  COS: {
    Bucket: 'your-bucket-name',
    Region: 'ap-guangzhou',
    // 临时密钥通过后端获取，或使用STS
  },
  SUPABASE: {
    URL: 'https://xxx.supabase.co',

  }

```

### likes.js - 点赞模块接口

```javascript
class LikesManager {
  constructor(supabaseClient);
  async getLikes(songId);           // 获取点赞数
  async toggleLike(songId, userId); // 切换点赞状态
  async hasUserLiked(songId, userId); // 检查用户是否已点赞
  onLikesChange(callback);         // 实时监听点赞变化
}
```

### upload.js - 上传模块接口

```javascript
class UploadManager {
  constructor(cosClient);
  async uploadSong(file, metadata);  // 上传MP3文件
  async uploadCover(file);           // 上传封面图片
  async getStsToken();               // 获取临时密钥
  onProgress(callback);             // 上传进度回调
}
```

## 实施注意事项

### 性能优化

- 上传时使用分片上传，支持大文件和断点续传
- 点赞数据使用LocalStorage缓存，减少API调用
- 图片使用缩略图，减少加载时间

### 用户体验

- 上传弹窗设计简洁，支持拖拽上传
- 点赞动画流畅，提供视觉反馈
- 上传失败时提供明确的错误提示

### 数据兼容

- 保留现有songs-config.json作为预设歌曲
- 新上传的歌曲存储在Supabase中
- 播放列表合并显示两种来源的歌曲
